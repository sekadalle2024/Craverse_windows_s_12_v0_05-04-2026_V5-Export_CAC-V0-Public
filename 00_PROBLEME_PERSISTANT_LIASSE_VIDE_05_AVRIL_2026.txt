```python
# Test avec UN SEUL poste pour vérifier le mapping
import openpyxl
from openpyxl import load_workbook

wb = load_workbook("Liasse_officielle_revise.xlsx")
ws = wb['ACTIF']

# Tester différentes cellules pour trouver la bonne
cellules_test = ['C10', 'C11', 'C12', 'C15', 'C20', 'E10', 'E11', 'E15']

print("Test des cellules dans ACTIF:")
for cellule in cellules_test:
    cell = ws[cellule]
    print(f"  {cellule}: {cell.value}")

# Chercher la cellule qui contient "Charges immobilisées" ou "AD"
print("\nRecherche de 'Charges immobilisées':")
for row in ws.iter_rows(min_row=1, max_row=50):
    for cell in row:
        if cell.value and 'Charges' in str(cell.value):
            print(f"  Trouvé en {cell.coordinate}: {cell.value}")
```

DIAGNOSTIC APPROFONDI: VÉRIFIER LES LOGS
================================================================================

Les logs ajoutés dans export_liasse.py devraient montrer:
  - "Données converties: X postes"
  - "ACTIF: X cellules remplies, Y erreurs"
  - "PASSIF: X cellules remplies, Y erreurs"

Si ces logs n'apparaissent pas:
  → Le backend n'a pas rechargé le fichier modifié
  → Solution: Redémarrer le backend

Si les logs montrent "0 cellules remplies":
  → Les données results sont vides
  → Solution: Vérifier la génération des données

Si les logs montrent "X cellules remplies" mais le fichier est vide:
  → Les mappings pointent vers les mauvaises cellules
  → Solution: Vérifier les mappings avec le template
