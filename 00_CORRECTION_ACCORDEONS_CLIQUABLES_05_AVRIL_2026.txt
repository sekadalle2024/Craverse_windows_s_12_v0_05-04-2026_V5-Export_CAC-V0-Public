PROBLEME IDENTIFIE
==================
Les 16 sections accordeon des etats de controle etaient bien affichees
en frontend, mais impossible de les ouvrir en cliquant dessus.

CAUSE RACINE
============
Le HTML est genere cote backend avec un attribut onclick inline:
```html
<div class="section-header" onclick="toggleSection(this)">
```

Mais la fonction toggleSection() etait definie dans une IIFE (fonction
auto-executee) dans EtatsControleAccordeonHandler.js, donc elle n'etait
PAS accessible dans le contexte global window.

Resultat: onclick="toggleSection(this)" ne trouvait pas la fonction
et ne faisait rien au clic.

SOLUTION APPLIQUEE
==================
Exposer la fonction toggleSection() globalement:

```javascript
// Dans EtatsControleAccordeonHandler.js
function toggleSection(header) {
    if (!header) return;
    const section = header.parentElement;
    if (!section) return;
    section.classList.toggle('active');
}

// AJOUT: Exposer globalement pour les attributs onclick
window.toggleSection = toggleSection;
```

Maintenant, quand le HTML backend utilise onclick="toggleSection(this)",
la fonction est trouvee dans window.toggleSection et fonctionne!

FONCTIONNEMENT
==============
1. Le script EtatsControleAccordeonHandler.js est charge au demarrage
2. Il definit toggleSection() et l'expose dans window.toggleSection
3. Le HTML backend genere des sections avec onclick="toggleSection(this)"
4. Au clic, le navigateur trouve window.toggleSection et l'execute
5. La classe 'active' est togglee sur la section
6. Le CSS fait la transition (max-height: 0 -> 5000px)

DOUBLE PROTECTION
=================
Le script offre maintenant DEUX mecanismes:

1. Attributs onclick inline (HTML backend)
   - onclick="toggleSection(this)" dans le HTML
   - Fonction window.toggleSection() disponible

2. Event listeners JavaScript (script frontend)
   - Attache des listeners sur .section-header
   - MutationObserver pour detecter les nouveaux elements
   - Reinitialisation automatique

Les deux mecanismes fonctionnent ensemble sans conflit!
