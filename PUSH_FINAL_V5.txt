⚡ SOLUTION ALTERNATIVE 1: SSH
═══════════════════════════════════════════════════════════════════════════

SSH est plus stable que HTTPS pour les gros transferts.

# 1. Générer une clé SSH
ssh-keygen -t ed25519 -C "votre-email@example.com"

# 2. Copier la clé publique
cat ~/.ssh/id_ed25519.pub

# 3. Ajouter sur GitHub: https://github.com/settings/keys

# 4. Changer l'URL
git remote set-url origin git@github.com:sekadalle2024/Claraverse_windows_s_11_v0_21-03-2026_V5-Public-Public-Public.git

# 5. Push
git push origin master


⚡ SOLUTION ALTERNATIVE 2: COMPRESSION RÉDUITE
═══════════════════════════════════════════════════════════════════════════

# Désactiver la compression
git config --global core.compression 0

# Push
git push origin master

# Réactiver après
git config --global core.compression -1


⚡ SOLUTION ALTERNATIVE 3: TIMEOUTS AUGMENTÉS
═══════════════════════════════════════════════════════════════════════════

# Augmenter tous les timeouts
git config --global http.postBuffer 1048576000
git config --global http.timeout 3600
git config --global http.lowSpeedLimit 0
git config --global http.lowSpeedTime 999999

# Réessayer
git push origin master
