========================================
DIAGNOSTIC - ACCORDEONS NON CLIQUABLES
========================================
Date: 05 Avril 2026
Statut: PROBLEME PERSISTANT

SYMPTOME
========
Les 16 sections accordeon des etats de controle s'affichent
mais ne s'ouvrent toujours pas au clic.

CORRECTIONS APPLIQUEES
=======================
1. Exposition de window.toggleSection = toggleSection
2. Suppression des attributs onclick pour eviter les conflits
3. Ajout de data-listenerAttached pour eviter les doublons
4. Amelioration du MutationObserver avec delai de 500ms
5. Logs detailles pour le debugging

TESTS A EFFECTUER
=================

1. VERIFIER LA CONSOLE DU NAVIGATEUR
   Ouvrir F12 et chercher:
   - "🔧 EtatsControleAccordeonHandler: Initialisation..."
   - "📊 Nombre de sections trouvées: X"
   - "✅ Listener ajouté sur section X"
   
   Si "Nombre de sections trouvées: 0":
   → Les sections ne sont pas encore dans le DOM
   → Attendre la generation complete des etats

2. VERIFIER LA STRUCTURE HTML
   Dans la console (F12), executer:

3. TESTER MANUELLEMENT
   Dans la console (F12), executer:

   window.toggleSection(header);
   
   // Verifier si la classe active est ajoutee

4. VERIFIER LES STYLES CSS
   Dans la console (F12), executer:

   console.log(window.getComputedStyle(content).maxHeight);
   
   // Apres avoir ajoute la classe active

   section.classList.add('active');
   console.log(window.getComputedStyle(content).maxHeight);

5. VERIFIER LES EVENT LISTENERS
   Dans la console (F12), executer:

PROBLEMES POSSIBLES
===================

A. LES SECTIONS NE SONT PAS DANS LE DOM
   Symptome: "Nombre de sections trouvées: 0"
   Solution: Attendre la generation complete ou reinitialiser
   Commande: window.reinitializeEtatsControleAccordeons();

B. LES STYLES CSS NE SONT PAS CHARGES
   Symptome: max-height ne change pas
   Solution: Verifier que etats_financiers.py contient les styles
   Fichier: py_backend/etats_financiers.py (lignes 1596-1750)

C. CONFLIT AVEC D'AUTRES SCRIPTS
   Symptome: Erreurs JavaScript dans la console
   Solution: Verifier l'ordre de chargement des scripts
   Fichier: index.html (ligne 151)

D. LE HTML EST GENERE APRES L'INITIALISATION
   Symptome: Logs d'initialisation avant les sections
   Solution: Le MutationObserver devrait detecter et reinitialiser
   Verifier: Logs "🆕 Nouvelles sections détectées"

E. LES ATTRIBUTS ONCLICK BLOQUENT LES LISTENERS
   Symptome: Clic ne fait rien
   Solution: Les attributs onclick sont maintenant supprimes
   Verifier: Logs "🔧 Attribut onclick supprimé"
