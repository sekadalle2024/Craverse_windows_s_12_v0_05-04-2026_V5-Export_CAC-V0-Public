Le workflow n8n "Programme contrôle comptes" retourne une structure spécifique:

{
  "Tableau entete": { ... },
  "Tableau processus": [ ... ]
}

La fonction convertStructuredDataToMarkdown() cherchait une clé contenant "tableau" 
(ligne 496-505), trouvait "Tableau entete", puis vérifiait si c'était un array.

Comme "Tableau entete" est un OBJET (pas un array), la fonction appelait 
convertGenericStructureToMarkdown() qui génère du texte brut.

═══════════════════════════════════════════════════════════════════════════════
  ✅ SOLUTION APPLIQUÉE
═══════════════════════════════════════════════════════════════════════════════

Ajout d'une détection spécifique AVANT la vérification array (ligne ~508):

1. Vérifier si data contient directement "Tableau entete" ET "Tableau processus"
2. Si OUI:
   - Traiter "Tableau entete" comme une table header
   - Traiter "Tableau processus" comme une table data_array
   - Retourner le Markdown généré
3. Si NON: continuer avec la logique existante

Fichier: src/services/claraApiService.ts

// 🔧 NOUVEAU FORMAT: Programme contrôle comptes avec "Tableau entete" et "Tableau processus"
const hasTableauEntete = "Tableau entete" in data;
const hasTableauProcessus = "Tableau processus" in data;

if (hasTableauEntete && hasTableauProcessus) {
  console.log("🔧 FORMAT PROGRAMME CONTROLE COMPTES DÉTECTÉ");
  
  // Traiter "Tableau entete" comme header
  const tableauEntete = data["Tableau entete"];
  if (tableauEntete && typeof tableauEntete === 'object') {
    markdown += this.convertHeaderTableToMarkdown(tableauEntete, 0);
  }
  
  // Traiter "Tableau processus" comme data_array
  const tableauProcessus = data["Tableau processus"];
  if (Array.isArray(tableauProcessus)) {
    markdown += this.convertArrayTableToMarkdown("Tableau processus", tableauProcessus);
  }
  
  return markdown;
}
