================================================================================
CORRECTION - APPLICATION AUTOMATIQUE DES COULEURS POUR TOUTES LES MATRICES
Date: 02 Avril 2026
================================================================================

PROBLÈME IDENTIFIÉ
================================================================================

Lorsque les matrices sont générées nativement via l'endpoint n8n, les codes
couleurs ne sont PAS appliqués automatiquement pour:

❌ Matrice alphabétique - 4 niveaux
❌ Matrice alphabétique - 5 niveaux  
❌ Matrice numérique - 3 niveaux
❌ Matrice numérique - 4 niveaux
❌ Matrice numérique - 5 niveaux

✅ Matrice alphabétique - 3 niveaux (FONCTIONNAIT)

L'action "Appliquer couleurs risques" du menu contextuel ne fonctionnait
que pour la matrice Alpha-3.

================================================================================
CAUSE RACINE
================================================================================

1. La méthode applyRiskStyle() utilisait getRiskColors() qui ne retournait
   que 3 couleurs (Faible, Moyen, Élevé)

2. La méthode applyRiskColors() normalisait toutes les valeurs vers ces
   3 niveaux via normalizeRiskValue()

3. Les valeurs des autres matrices (Mineur, Significatif, Majeur, Critique,
   Très faible, Très élevé, valeurs numériques 1-25) n'étaient pas reconnues

================================================================================
SOLUTION APPLIQUÉE
================================================================================

1. MODIFICATION DE applyRiskStyle()
   ---------------------------------
   
   Avant:
   ------
   - Utilisait getRiskColors() (3 couleurs seulement)
   - Normalisait via normalizeRiskValue() (3 niveaux)
   - Ne reconnaissait que: Faible, Moyen, Élevé
   
   Après:
   ------
   - Détection AUTOMATIQUE du type de valeur
   - Support de TOUTES les matrices:
     * Alpha-3: Faible, Moyen, Élevé
     * Alpha-4: Mineur, Significatif, Majeur, Critique
     * Alpha-5: Très faible, Faible, Modéré, Élevé, Très élevé
     * Num-3: Valeurs 1-9 avec couleurs appropriées
     * Num-4: Valeurs 1-16 avec couleurs appropriées
     * Num-5: Valeurs 1-25 avec couleurs appropriées

2. MODIFICATION DE applyRiskColors()
   ----------------------------------
   
   Avant:
   ------
   const normalizedValue = this.normalizeRiskValue(value);
   if (normalizedValue) {
     this.applyRiskStyle(cell, normalizedValue);
   }
   
   Après:
   ------
   if (value) {
     // Passer la valeur brute - applyRiskStyle détecte automatiquement
     this.applyRiskStyle(cell, value);
   }

================================================================================
DÉTAILS TECHNIQUES - applyRiskStyle()
================================================================================

ORDRE DE DÉTECTION (important pour éviter les conflits):

1. Matrices Alpha-5 (détection en premier car plus spécifique)
   - "Très faible" / "très faible" → Vert foncé (#28a745)
   - "Très élevé" / "très élevé" → Rouge (#dc3545)
   - "Modéré" / "modéré" → Jaune (#ffc107)

2. Matrices Alpha-4
   - "Mineur" / "minor" → Vert (#28a745)
   - "Significatif" / "significant" → Jaune (#ffc107)
   - "Majeur" / "major" → Orange (#ff8c00)
   - "Critique" / "critical" → Rouge (#dc3545)

3. Matrices Alpha-3 (après Alpha-5 pour éviter confusion)
   - "Faible" / "low" → Vert (#28a745)
   - "Moyen" / "medium" → Jaune (#ffc107)
   - "Élevé" / "high" → Rouge (#dc3545)

4. Matrices Numériques
   - Valeurs 1-9: Utilise getColorsNumeric(9)
   - Valeurs 10-16: Utilise getColorsNumeric(16)
   - Valeurs 17-25: Utilise getColorsNumeric(25)
   - Valeurs 1-5: Couleurs de base (vert → rouge)

NOTES IMPORTANTES
================================================================================

1. L'ordre de détection dans applyRiskStyle() est CRITIQUE
   - "Très faible" doit être détecté AVANT "Faible"
   - "Très élevé" doit être détecté AVANT "Élevé"
   - Sinon, "Très faible" serait détecté comme "Faible"

2. Les valeurs numériques 1-5 peuvent appartenir à n'importe quelle matrice
   - Impossible de distinguer sans contexte
   - Utilisation de couleurs de base (vert → rouge)

3. La méthode est case-insensitive et supporte les accents
   - "Très faible" = "très faible" = "tres faible"
   - "Élevé" = "élevé" = "eleve"
